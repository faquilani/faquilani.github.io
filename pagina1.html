Configurazione della Rete

Descrizione della Rete

Il progetto prevede una rete scolastica distribuita su tre sedi con routing statico tra i router e connessioni seriali tra di essi. Ogni sede è segmentata in più sottoreti, utilizzando VLAN per separare la rete amministrativa dalla rete didattica.

Sedi e Configurazione della Rete

Sede 1 (Scuola 1)

  • Rete WLAN Aule/Biblioteca:
    • Gateway: 192.168.0.1
    • DNS: 8.8.8.8
  • Rete Segreteria:
    • Gateway: 192.168.1.1
    • DNS: 8.8.8.8
  • Router Sede 1:
    • Porta seriale verso Sede 2: 200.100.50.1
    • Porta seriale verso Server (Sede 3): 200.200.50.1

Sede 2 (Scuola 2)

  • Rete WLAN Aule/Biblioteca:
    • Gateway: 192.168.2.1
    • DNS: 8.8.8.8
  • Rete Segreteria:
    • Gateway: 192.168.4.1
    • DNS: 8.8.8.8
  • Router Sede 2:
    • Porta seriale verso Sede 1: 200.100.50.2
    • Porta seriale verso Server (Sede 3): 200.200.50.2

Sede 3 (Centro Server e DHCP)

  • Rete Server:
    • Gateway: 192.168.3.1
    • Server DNS/DHCP: 192.168.3.2
  • Router Sede 3:
    • Porta seriale verso Sede 1: 200.200.50.1
    • Porta seriale verso Sede 2: 200.200.50.2

Configurazione IP & Routing

Il routing statico è stato utilizzato per permettere la comunicazione tra le sedi.

Configurazione dei Router

enable
configure terminal

! Configurazione interfacce LAN per le sottoreti
interface GigabitEthernet0/0
ip address 192.168.0.1 255.255.255.0
no shutdown

interface GigabitEthernet0/1
ip address 192.168.1.1 255.255.255.0
no shutdown

! Configurazione interfacce seriali
interface Serial0/0/0
ip address 200.100.50.1
255.255.255.252
clock rate 64000
no shutdown

interface Serial0/0/1
ip address 200.200.50.1 255.255.255.252
no shutdown

! Configurazione del routing statico
ip route 192.168.2.0 255.255.255.0 200.100.50.2
ip route 192.168.4.0 255.255.255.0 200.100.50.2
ip route 192.168.3.0 255.255.255.0 200.200.50.2

! Configurazione DHCP Helper per inoltrare richieste DHCP al server remoto
interface GigabitEthernet0/0
ip helper-address 192.168.3.2

interface GigabitEthernet0/1
ip helper-address 192.168.3.2

Configurazione degli Switch (Trunking per VLAN WLAN e Segreteria)

Sugli switch che gestiscono le sottoreti delle scuole, il trunking è stato abilitato per permettere il traffico di VLAN multiple.

enable
configure terminal

! Configurazione porte trunk per supportare VLAN multiple
interface GigabitEthernet0/1
switchport mode trunk

interface GigabitEthernet0/2
switchport mode trunk

Conclusione

Questa rete permette una comunicazione sicura ed efficiente tra le sedi, garantendo la separazione logica delle sottoreti tramite VLAN e un instradamento gestito con routing statico. L'uso del server DHCP centralizzato semplifica l'assegnazione degli indirizzi IP, mentre il trunking sugli switch consente la gestione delle WLAN interne.
